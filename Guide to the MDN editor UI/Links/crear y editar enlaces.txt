Afortunadamente, aunque haya <em>muchas </em>maneras de crear&nbsp;los vinculos, estos son muy faciles de hacer.</span></p>

<h2 id="Usando_la_barra_de_herramientas">Usar la barra de herramientas</h2>

<p>La forma mas obvia de crear un&nbsp;vinculo es hacer clic en el boton "vinculo" en&nbsp;la&nbsp;&nbsp;barra&nbsp; de herramientas, o presionar <kbd>Ctl</kbd> +<kbd>K</kbd>&nbsp;(<kbd>Command</kbd> +<kbd>K</kbd>&nbsp;en MacOs).&nbsp;&nbsp;El boton&nbsp;vinculo se ve asi:&nbsp;<img alt="el boton enlace 
(usado desde 4/mayo/2015)" src="https://mdn.mozillademos.org/files/12003/link-
button.png" style="height:16px; width:16px" />. Puedes usar esta caracteristica para crear un&nbsp;vinculo desde cero, o para añadir un&nbsp;vinculo a texto ya existente.</p>

<h3 id="Crear_vinculo_desde_cero">Crear&nbsp;vinculo desde cero</h3>

<p>Una vez que hayas hecho clic en el botón vinculo, aparecerá la ventana de edicion de vinculo, como se muestra aqui abajo:</p>

<p><img alt="menu editor cinculos" src="https://mdn.mozillademos.org/files/16003/es.JPG" style="height:329px; width:429px" /></p>

<p>Aqui puedes crear el nuevo vinculo; los campos en esta ventana son:</p>

<dl>
 <dt>Tipo de vinculo&nbsp;</dt>
 <dd>Es el tipo de vinculo que vas a crear. El valor por defecto es URL (Localizador uniforme de recursos), la URL es para especificar una direccion web, ya sea en la MDN o en cualquier sitio web externo.<br />
 Tambien puedes elegir las opciones "Referencia en esta pagina" o "E-mail", la opcion "Referencia en esta pagina"&nbsp; te permite crear un vinculo a una referencia (anchor) que hayas insertado anteriormente en la pagina usando el boton <strong>Referencia</strong> <img alt="anchor button" src="https://mdn.mozillademos.org/files/16006/anchor.JPG" style="height:18px; width:25px" />&nbsp;de la barra de herramientas,&nbsp;si no has insertado ninguna referencia te mostrara un mensaje recordandotelo. La opcion E-mail te permite insertar un vinculo <code>mailto:</code> para que el lector pueda enviar un correo (se abre una nueva ventana para el lector) a la direccion de correo electronico que especifiques; tienes la opcion de añadirle titulo y cuerpo. Casi siempre utilizaras la opcion URL.</dd>
 <dt>Busqueda del titulo del articulo / Texto del vinculo</dt>
 <dd>Este campo tiene dos propositos: el primero es permitirte especificar el texto del vinculo si no quieres que se muestre la URL completa; el segundo es buscar articulos en MDN que coincidan con el texto que pongas en este campo, para crear un vinculo a ellos. Por ejemplo, si escribes "JavaScript" en este campo, veras algo como esto:</dd>
 <dd><img alt="" src="https://mdn.mozillademos.org/files/16005/Rollup

%20menu.JPG" style="height:340px; width:557px" /></dd>
</dl>

<dl>
 <dd>Aqui puedes ver una lista de titulos de paginas en MDN que incluyen el texto que escribiste. Puedes elegir una de ellas, o seguir escribiendo para encontrar otras opciones. Ten en cuenta que las opciones de la lista indican el idioma ("[es]" en este caso). El texto que indica el idioma "[es]" no aparece en el texto del vinculo; solo aparece para ayudarte a saber que estas creando un vinculo a un articulo de MDN que esta disponible en el mismo idioma que estas editando.</dd>
 <dt>Archivos adjuntos</dt>
 <dd>Tambien puedes hacer que el vinculo sea un enlace a algun archivo que se haya adjuntado a la pagina seleccionandolo en la opcion archivos adjuntos. Esta es una excelente manera para compartir codigo fuente (Source code), imagenes, o cualquier otro archivo para que el lector lo descargue.</dd>
 <dt>URL</dt>
 <dd>Por ultimo, el campo URL te deja escribir directamente (o copiar/pegar) la URL; si elegiste un articulo de MDN en el campo <strong>Busqueda del titulo del articulo / Texto del vinculo</strong> o seleccionaste un archivo en el campo <strong>Archivos adjuntos</strong> se mostrara la URL de estos. Es algo comun copiar y pegar las URLs de otras paginas en MDN que estes revisando, sin embargo el nombre del dominio (https://developer.mozilla.org) se debe borrar ya que no es necesaria (por ejemplo <s>https://developer.mozilla.org</s>/es/docs/MDN/Contribute/Editor).</dd>
</dl>

<p>Cuando el vinculo este configurado, has clic en el boton: <img alt="ok button [es]" src="https://mdn.mozillademos.org/files/16007/boton%20aceptar.JPG" style="height:28px; width:70px" />&nbsp;para insertarlo.</p>

<div class="note">
<p>Existe una segunda pestaña—<strong>Avanzado</strong>—en la ventana del editor de vinculo. Por el momento, no te aconsejaremos usar alguna&nbsp;de las opciones en esa pestaña. Es posible que en el futuro existan diferentes estilos para los vinculos, pero es probable que se añadan nuevos widgets a la barra de herramientas para usar esas caracteristicas cuando esten disponibles.</p>
</div>

<h3 id="Vincular_texto_existente">Vincular texto existente</h3>

<p>Si hay alguna porcion de texto que quieras convertir a un vinculo, puedes simplificar el proceso un poco. Selecciona la porcion de texto que quieras convertir a un vinculo antes de abrir la ventana del editor de vinculos; esto añadira el texto seleccionado al campo <strong>Busqueda del articulo / Texto del vinculo</strong>. Por ejemplo; digamos que tenemos el siguiente texto</p>

<blockquote>
<p>Anita lava la tina JavaScript la tina lava Anita</p>
</blockquote>

<p>Y nosotros queremos que "JavaScript" sea un vinculo que nos enlace a una página relacionada con JavaScript. Solo selecciona esa palabra y abre el editor de vínculos. Al colocar el cursor sin hacer clic sobre alguna opcion, podras ver su "slug" (parte de la direccion URL que aparece despues del nombre de dominio), el cual te puede dar una mejor idea de donde se localiza el artículo y que tipo de artículo es.</p>

<p><img alt="" src="https://mdn.mozillademos.org/files/16004/slug%20es.jpg" style="height:338px; width:555px" /></p>

<p>Aqui puedes buscar y elegir la opcion que mejor te paresca. Al elegir cualquier opcion (por ejemplo "Acerca de JavaScript [es]") automaticamente se llenara el campo de la ULR y podras simplemente hacer clic en el boton <img alt="ok button [es]" src="https://mdn.mozillademos.org/files/16007/boton%20aceptar.JPG" style="height:29px; width:71px" />&nbsp;y convertir el texto seleccionado en un vinculo, como a continuacion:</p>

<blockquote>
<p>Anita lava la tina <a href="/es/docs/Web/JavaScript/Acerca_de_JavaScript">JavaScript</a>&nbsp;la tina lava Anita.</p>
</blockquote>

<h2 id="Usar_macros_de_vinculos">Usar macros de vinculos</h2>

<p>MDN utiliza mucho las macros&nbsp;para crear vinculos automaticamente y apropiadamente de un termino, al mismo tiempo que le da el estilo correcto al vinculo deacuerdo con nuestra guia de estilo. Toma en cuenta esto: nuestra guia de estilo indica que los terminos API, los elementos y atributos HTML, las propiedades CSS, el nombre de metodos, y parecidos, se les debe dar el estilo con el elemento: {{HTMLElement("code")}}. Tambien, usualmente, deben ser vinculos que enlacen a la respectiva pagina en MDN</p>

<p>Toma algo de tiempo acostumbrarse a crear vinculos usando macros, pero tiene varias ventajas:</p>

<ul>
 <li>Aplica el estilo apropiado por ti.</li>
 <li>Crea el vinculo por ti—y este es aprueba de futuros cambios en la estructura de MDN.</li>
 <li>Tambien puede crear descripciones emergentes (tooltips) para tí.</li>
</ul>

<p>Existen demasiadas macros&nbsp;y nosotros no las veremos todas aqui. En vez de eso, veremos algunos ejemplos de las mas comunes. Para ver la lista completa, mira la seccion "<a href="/en-
US/docs/Project:MDN/Contributing/Custom_macros#Creating_hyperlinks" title="/en-
US/docs/Project:MDN/Contributing/Custom_macros#Creating_hyperlinks">Creating hyperlinks</a>" (no disponible en español actualmente) en nuestra guía <a href="/en-
US/docs/Project:MDN/Contributing/Custom_macros" title="/en-
US/docs/Project:MDN/Contributing/Custom_macros">Custom macros for MDN</a>. No cuesta nada revisar el codigo fuente de&nbsp;<a href="/en-
US/docs/Project:Introduction_to_KumaScript" title="/en-
US/docs/Project:Introduction_to_KumaScript">KumaScript</a>&nbsp;(vinculo no disponible en español) para encontrar cualquier macro; la mayoria de estas tienen comentarios que explican como funcionan y cuales son sus parametros, si es que tiene.</p>

<h3 id="Crear_vinculos_a_la_documentacion_para_APIs">Crear vinculos a la documentacion para APIs.</h3>

<p>Tenemos un numero de macros que resultan de mucha ayuda para crear vinculos con el estilo apropiado para las APIs. Aqui presentamos algunas de las mas utiles;&nbsp;en cada caso, talvez existan parametros que puedas añadir para tener mas control sobre lo que se va a mostrar (como por ejemplo: evitar que se aplique el estilo del elemento:&nbsp;<code>&lt;code&gt;</code>&nbsp;). Puedes hacer clic en cualquier nombre de macro de las 4 que se enlistan aqui abajo, para leer el codigo fuente de la macro (solo en ingles); todas las macros&nbsp;tienen comentarios al inicio que explican su funcionamiento y cuales son sus&nbsp;parametros.</p>

<dl>
 <dt>{{TemplateLink("HTMLElement")}}</dt>
 <dd>Inserta el nombre de un elemento HTML, con el vinculo y estilo apropiado. Por ejemplo: <code>\{{HTMLElement("table")}}</code> se convierte en: {{HTMLElement ("table")}}.</dd>
 <dt>{{TemplateLink("cssxref")}}</dt>
 <dd>Inserta una propiedad CSS, una regla-at (at = @), o la documentacion de un selector en la referencia CSS. Por ejemplo: <code>\{{cssxref("background- color")}} </code> te da como resultado: {{cssxref("background-color")}}.</dd>
 <dt>{{TemplateLink("domxref")}}</dt>
 <dd>Inserta un vinculo hacia la referencia API web de un termino API. Por ejemplo: <code>\{{domxref("window")}}</code> se convierte en: {{domxref ("window")}} y <code>\{{domxref("window.scrollBy()")}}</code> se convierte en: {{domxref ("window.scrollBy()")}}.</dd>
 <dt>{{TemplateLink("SVGElement")}}</dt>
 <dd>Inserta el nombre de un elemento SVG, con el vinculo y estilo correcto. Por ejemplo: <code>\{{SVGElement("circle")}}</code> se convierte en: {{SVGElement ("circle")}}.</dd>
</dl>

<h3 id="Crear_vinculo_a_una_seccion_en_el_mismo_articulo">Crear vinculo a una seccion en el mismo articulo</h3>

<p>Para crear un vinculo que te diriga a una seccion dentro del mismo articulo/pagina, puedes usar la macro: {{TemplateLink("anch")}} La sintaxis es muy clara: <code> \{{anch ("Nombre de la seccion")}}</code>. Por defecto, el texto del vinculo es el titulo de la seccion, pero puedes agregar un segundo parametro (opcional) que sirve para mostrar el texto del segundo parametro en vez del titulo de la seccion. Aqui un ejemplo:</p>

<ul>
 <li>\{{anch("Usar la barra de herramientas")}} se ve así: {{anch("Usar la barra de herramientas")}}</li>
 <li>\{{anch("Usar la barra de herramientas", "anteriormente")}} se ve así: {{anch("Usar la barra de herramientas", "anteriormente")}}</li>
</ul>

<h3 id="Crear_vinculos_hacia_bugs">Crear vinculos hacia bugs</h3>

<p>Puedes crear un vinculo a un bug en la base de datos de Mozilla con la macro: {{TemplateLink("bug")}} Esta macro acepta solo un parametro: el numero del bug al cual queremos dirigirnos. Por ejemplo, <code>\{{bug (765642)}}</code> se ve así: {{bug (765642)}}.</p>

<p>Del mismo modo, puedes crear vinculos a bugs de otros navegadores web:</p>

<dl>
 <dt>Microsoft Edge</dt>
 <dd>{{TemplateLink("EdgeBug")}}: <code>\{{edgebug(10569045)}}</code> se convierte en {{edgebug(10569045)}}.</dd>
 <dt>Google Chrome</dt>
 <dd>{{TemplateLink("ChromeBug")}}: <code>\{{chromebug(31277)}}</code> se convierte en {{chromebug(31277)}}.</dd>
 <dt>WebKit (Safari, etc.)</dt>
 <dd>{{TemplateLink("WebkitBug")}}: <code>\{{webkitbug(31277)}}</code> se convierte en {{webkitbug(31277)}}.</dd>
</dl>

<h3 id="Crear_vinculos_a_RFCs">Crear vinculos a RFCs</h3>

<p>Mucho de la raiz del funcionamiento de la internet esta documentado en los RFCs (Request For Comments). Puedes facilmente hacer una referencia a un RFCs usando la macro: {{TemplateLink("RFC")}} Por ejemplo, \{{RFC(2616)}} se convierte en: {{RFC (2616)}}. Opcionalmente puedes añadir texto que sirva como texto del vinculo y/o tambien puedes añadir el numero de la seccion dentro del articulo RFC al cual se va a dirigir el vinculo.</p>

<h3 id="Crear_vinculo_a_informacion_acerca_de_las_interfaces_XPCOM">Crear vinculo a informacion acerca de las interfaces XPCOM</h3>

<div class="note">
<p>El staff de MDN ya no se encarga de mantener la documentacion de XPCOM, pero las contribuciones voluntarias son bienvenidas.</p>
</div>

<p>Si estas documentando partes internas de Mozilla, la capacidad de crear vinculos a la documentacion de las interfaces XPCOM es de mucha ayuda. Existen algunas macros para esto.</p>

<p>La sintaxis para crear un vinculo a toda la informacion de una interface es solamente la siguiente: <code>\{{interface("nombre de la interface")}}</code>. Por ejemplo, si escribes:</p>

<blockquote>
<p>Cuando necesites analizar o crear URIs, la interface: \{{interface ("nsIIOService")}} te puede ayudar.</p>
</blockquote>

<p>El resultado se vera así:</p>

<blockquote>
<p>Cuando necesites analizar o crear URIs, la interface: {{interface ("nsIIOService")}} te puede ayudar.</p>
</blockquote>

<p>Si necesitas crear un vinculo a la informacion de un metodo o un atributo especifico de una interfaz XPCOM, la macro: {{TemplateLink("ifmethod")}} y la macro: {{TemplateLink("ifattribute")}} son para tí. Estas aceptan como parametros el nombre de la interfaz y el nombre del metodo o atributo al cual quieres referenciar. La macro: {{TemplateLink("ifmethod")}} es particularmente interesante, ya que crea un formato especial que añade los parentesis obligados por la guia de estilo despues del nombre del metodo. Por ejemplo, <code> \{{ifmethod("nsIIOService", "newURI")}}</code> se convierte en {{ifmethod ("nsIIOService", "newURI")}}. Este es uno de los casos en el que eres protegido de posibles cambios futuros en la guia de estilo.</p>

<h3 id="Crear_vinculo_a_la_documentacion_de_las_preferencias_de_Mozilla">Crear vinculo a la documentacion de las preferencias de Mozilla</h3>

<p>Para insertar el nombre de una preferencia Mozilla y hacer que se enlace con la pagina que le corresponde en la <a href="/en-

US/docs/Mozilla/Preferences/Preference_reference" title="/en-

US/docs/Mozilla/Preferences/Preference_reference">Preference reference</a> (no disponilbe en español todavia) usa la macro: {{TemplateLink("pref")}} Esta macro acepta un parametro: el nombre completo de la preferencia al cual quieres enlazar. Por ejemplo,puedes usar esto: <code>\{{pref ("javascript.options.showInConsole")}}</code> para crear esto: {{pref ("javascript.options.showInConsole")}}.</p>

<h3 id="Crear_vinculo_a_un_archivo_de_codigo_fuente_de_Mozilla">Crear vinculo a un archivo de codigo fuente de Mozilla</h3>

<p>Puedes crear vinculos a archivos de codigo fuente de Mozilla (aunque probablemente no lo hagas muy seguido) usando la macro {{TemplateLink ("source")}}. En vez de especificar la URL completa del archivo, puedes simplemente especificar una direccion en la carpeta <code>/source/</code> sin escribir todo lo que le antecede. Por ejemplo: \{{source("browser/Makefile.in")}} crea este vinculo: {{source ("browser/Makefile.in")}} (la direccion completa seria: https://dxr.mozilla.org/mozilla- central/source/browser/Makefile.in).</p>

<p>Opcionalmente, tambien puedes especificar el texto del vinculo con el que se va a mostrar el vinculo. Por ejemplo, puedes escribir esto: \{{source ("browser/Makefile.in", "el archivo makefile de la carpeta browser")}} para que se muestre esto:: {{source ("browser/Makefile.in", "el archivo makefile de la carpeta browser")}}.</p>

<h2 id="Crear_vinculo_a_contenido_recomendado">Crear vinculo a contenido recomendado</h2>

<p>Si quieres crear una lista de paginas relacionadas, o de algun otro material de lectura recomendado, puedes hacerlo creando un grupo de "quicklinks" en la barra lateral que se muestra al inicio de la pagina; este mecanismo reemplasa nuestra seccion "vease tambien" que se mostraba al final del articulo. Los detalles acerca de como crear grupos de "quicklinks", visita: <a href="/en-

US/docs/MDN/Contribute/Structures/Quicklinks">Quicklinks</a>(no disponible en español todavia).</p>

<h2 id="Esquemas_URL">Esquemas URL</h2>

<p>Por razones de seguridad, solo debes crear vinculos que usen los siguientes esquemas:</p>

<ul>
 <li><code>http://</code></li>
 <li><code>https://</code></li>
 <li><code>ftp://</code></li>
 <li><code>mailto:</code></li>
</ul>

<p>Tal vez otros esquemas puedan funcionar o no, pero no estan aprobados y probablemente sean eliminados por el staff editorial.</p>

<div class="note">
<p>Esquemas especiales de URL tales como <code>about:</code> y <code>chrome:</code> son usados por Firefox, Google Chrome, y otros navegadores para dar acceso a contenido especial como las preferencias, informacion acerca de debugging, y mas. Esos vinculos no funcionan dentro de los articulos de MDN, por lo tanto no intente crear vinculos usando esos esquemas. Lo mismo aplica para los esquemas: <code>javascript:</code> y <code>jar:</code> que estan bloqueados por la mayoria de los navegadores modernos como medida de precaucion.</p>
</div>
